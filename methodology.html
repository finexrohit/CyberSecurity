Methodology

Step-by-Step Approach to Building the Detection System

The development of the Cyberbullying Detection System follows a systematic methodology that encompasses data collection, preprocessing, feature extraction, model training, evaluation, and deployment. Each phase is crucial for building an accurate and reliable classification system.

1

Data Collection

The first step involves gathering a comprehensive dataset of labeled text samples containing both cyberbullying and non-bullying content.

Data Sources

  • Kaggle Datasets: Publicly available cyberbullying detection datasets
  • Twitter API: Social media posts (with appropriate permissions)
  • Research Datasets: Academic datasets from published studies
  • Manual Annotation: Expert-labeled samples for validation

Dataset Characteristics

~50,000+ Text Samples
2-3 Classes
80:20 Train:Test Split

Sample Data Format

| Text                                    | Label        |
|-----------------------------------------|--------------|
| "You're amazing and I love your work!"  | Non-Bullying |
| "You're so ugly, nobody likes you"      | Bullying     |
| "This could be better, try again"       | Non-Bullying |
2

Text Preprocessing

Raw social media text contains noise that must be cleaned before feeding it to the machine learning model. Preprocessing transforms unstructured text into a standardized format.

Preprocessing Pipeline

2.1
Lowercasing

Convert all text to lowercase for consistency

"HELLO World" ‚Üí "hello world"
2.2
URL Removal

Remove hyperlinks and web addresses

"Check https://example.com" ‚Üí "Check "
2.3
Mention & Hashtag Removal

Remove @mentions and #hashtags

"@user #topic hello" ‚Üí " hello"
2.4
Special Character Removal

Remove punctuation, emojis, and numbers

"Hello!!! 123" ‚Üí "Hello"
2.5
Tokenization

Split text into individual words (tokens)

"hello world" ‚Üí ["hello", "world"]
2.6
Stopword Removal

Remove common words with little meaning

["the", "is", "good"] ‚Üí ["good"]
2.7
Lemmatization

Reduce words to their base/dictionary form

"running", "runs" ‚Üí "run"
3

Feature Extraction

Machine learning models require numerical input. Feature extraction converts preprocessed text into numerical vectors that capture the semantic meaning of the text.

TF-IDF Vectorization

Term Frequency-Inverse Document Frequency (TF-IDF) is used to convert text into numerical features. It measures how important a word is to a document relative to the entire corpus.

TF-IDF Formula

TF-IDF(t,d) = TF(t,d) √ó IDF(t)

Where:

  • TF(t,d) = Frequency of term t in document d
  • IDF(t) = log(Total documents / Documents containing t)

Feature Vector Example

Input: "you are ugly stupid"
                        
After TF-IDF Vectorization:
[0.0, 0.0, 0.45, 0.0, 0.52, 0.0, 0.0, 0.48, ...]
                        
(Sparse vector with weights for each word in vocabulary)
4

Model Training

Multiple machine learning algorithms are trained and compared to identify the best performing model for cyberbullying detection.

Algorithms Evaluated

üéØ Support Vector Machine (SVM)

Effective for high-dimensional text data, finds optimal hyperplane for classification

üå≤ Random Forest

Ensemble method using multiple decision trees, robust against overfitting

üìä Naive Bayes

Probabilistic classifier, fast training and prediction, good baseline

üìà Logistic Regression

Simple yet effective, provides probability scores, interpretable

Training Process

  1. Split data into training (80%) and testing (20%) sets
  2. Apply TF-IDF vectorization to training data
  3. Train each algorithm on the training set
  4. Perform cross-validation (k=5) for robust evaluation
  5. Tune hyperparameters using Grid Search
  6. Select best performing model
5

Model Evaluation

The trained models are evaluated using standard classification metrics to assess their performance and select the best model.

Evaluation Metrics

Accuracy

Overall correct predictions / Total predictions

Target: ‚â• 85%
Precision

True Positives / (True Positives + False Positives)

Target: ‚â• 80%
Recall

True Positives / (True Positives + False Negatives)

Target: ‚â• 80%
F1-Score

Harmonic mean of Precision and Recall

Target: ‚â• 80%

Confusion Matrix

Actual: Non-Bullying True Negative (TN) False Positive (FP)
Actual: Bullying False Negative (FN) True Positive (TP)
6

Deployment

The final trained model is serialized and integrated into a Flask web application for real-time cyberbullying detection.

Deployment Steps

  1. Model Serialization: Save trained model using pickle (.pkl format)
  2. Flask Integration: Load model in Flask application
  3. API Development: Create prediction endpoint
  4. Frontend Connection: Connect web interface to backend
  5. Testing: Validate end-to-end functionality
  6. Local Deployment: Run on localhost for demonstration
