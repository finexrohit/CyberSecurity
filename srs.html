Software Requirement Specification

Technical Requirements and System Architecture

1. Document Overview

1.1 Purpose

This Software Requirement Specification (SRS) document provides a complete description of all the functionalities and specifications of the Cyberbullying Detection System. It serves as a guide for development, testing, and deployment phases.

1.2 Scope

The system is designed to detect cyberbullying content in social media text using machine learning. It provides a web-based interface for users to input text and receive classification results indicating whether the content contains cyberbullying.

1.3 Definitions and Acronyms

Term Definition
NLP Natural Language Processing
ML Machine Learning
TF-IDF Term Frequency-Inverse Document Frequency
API Application Programming Interface
SVM Support Vector Machine

2. Functional Requirements

FR-01

Text Input

Description: The system shall accept text input from users through a web form.

  • Support for single text entries (up to 5000 characters)
  • Text area with placeholder instructions
  • Submit button to initiate analysis
  • Clear/Reset button to clear input
FR-02

Text Preprocessing

Description: The system shall preprocess input text using NLP techniques.

  • Convert text to lowercase
  • Remove URLs, mentions, and hashtags
  • Remove special characters and numbers
  • Tokenize text into words
  • Remove stopwords
  • Apply lemmatization
FR-03

Text Classification

Description: The system shall classify preprocessed text using ML model.

  • TF-IDF vectorization of cleaned text
  • Classification using trained ML model
  • Output categories: Bullying, Non-Bullying, Potentially Harmful
  • Confidence score calculation
FR-04

Result Output

Description: The system shall display classification results to the user.

  • Clear visual indication of classification (color-coded)
  • Confidence score as percentage
  • Descriptive message explaining the result
  • Option to analyze new text
